IoT Smart Garden System

Automated Irrigation & Monitoring with ESP32-C3 & RainMaker

1. Project Overview

The IoT Smart Garden is an automated plant watering system designed to maintain optimal soil moisture levels for plants. Built on the powerful ESP32-C3 microcontroller, it integrates local sensing, real-time visual feedback via an OLED display, and secure remote management through the ESP RainMaker cloud platform.

Key Innovation: The system features seamless integration with Voice Assistants (Alexa and Siri), allowing users to check plant status and control irrigation completely hands-free.

2. Hardware Design

2.1 System Architecture

The hardware design follows a modular approach, interfacing input sensors and output actuators with the central processing unit:

  • Input: Capacitive Soil Moisture Sensor (Analog ADC), Tactile Push Button (Digital Input).
  • Output: 5V Relay Module (Controls Water Pump), SSD1306 OLED Display (I2C), Status LED.
  • Power: 5V USB power supply regulating voltage for the ESP32-C3 and peripherals.

2.2 Schematic & Pin Configuration

The hardware interfacing logic is encapsulated in the plant_driver.h and oled_driver.h headers:

Component ESP32-C3 Pin Interface Description
Water Pump (Relay) GPIO 2 Digital Out Active High trigger for water pump.
Moisture Sensor GPIO 4 Analog (ADC1) Reads soil resistance values (High = Dry).
Manual Button GPIO 5 Digital In Active Low (Pull-up) trigger for manual watering.
Status LED GPIO 8 Digital Out Indicates system heartbeat and activity.
OLED SDA GPIO 10 I2C SDA Data line for display communication.
OLED SCL GPIO 9 I2C SCL Clock line for display communication.

3. Software Architecture

The firmware is developed using ESP-IDF v5.5.1 and leverages FreeRTOS for efficient multitasking. The codebase is structured to separate hardware abstraction layers (HAL) from the core application logic.

3.1 Technology Stack

  • Operating System: FreeRTOS (Real-Time Operating System).
  • Cloud Platform: ESP RainMaker (AWS-based Serverless IoT).
  • Provisioning: Wi-Fi Provisioning via BLE/SoftAP (Security 1).

3.2 Directory Structure

  • app_main.c: System entry point. Initializes NVS, Drivers, Wi-Fi stack, and RainMaker agent.
  • app_logic.c: Implements the core business logic, tasks, and global state management.
  • plant_driver.c: HAL for the water pump relay and moisture sensors.
  • oled_driver.c: HAL for the SSD1306 OLED display using ESP-IDF I2C Master API.

4. RTOS and Multitasking Implementation

The system utilizes FreeRTOS tasks to manage concurrent operations, ensuring that network communication does not block sensor readings or user input.

4.1 Task Definition

Four primary tasks are initialized in app_tasks_start():

  1. SensorControlTask (Priority 5): Reads ADC values, evaluates "Auto-Watering" logic, and updates Cloud parameters.
  2. ManualButtonTask (Priority 5): Polls the physical button state with software debouncing logic to trigger immediate watering.
  3. OLEDUpdateTask (Priority 3): Refreshes the display interface every 2 seconds with connection status and sensor data.
  4. HeartbeatTask (Priority 2): Blinks the LED to indicate system health and prevent watchdog timeouts.

4.2 Inter-Task Communication

  • Global State (system_state_t): A shared data structure holds sensor values (`soil_moisture_raw`, `temperature`) and pump status (`is_pumping`).
  • Mutex (g_state_mutex): A FreeRTOS Semaphore protects the shared state. It prevents race conditions between the Sensor task (writing data), OLED task (reading data), and Cloud callbacks (controlling actuators).

5. Functionality and Cloud Integration

5.1 Cloud Features (ESP RainMaker)

The system creates a virtual device node exposing standard parameters:

  • Power (Switch): Remote toggle for the water pump.
  • Soil Moisture (Sensor): Real-time soil data visualization.
  • Temperature & Humidity: Environmental monitoring.

5.2 Voice Assistant Integration

The system supports advanced voice control via cloud-to-cloud integration:

  • Amazon Alexa: The water pump is exposed as a Smart Switch. Users can say "Alexa, turn on the Water Pump" or ask "Alexa, what is the temperature in the Smart Garden?".
  • Apple Siri (iOS Shortcuts): Integrated with Siri Shortcuts via the RainMaker iOS app. Custom phrases like "Hey Siri, water the plants" trigger specific actions directly from the iPhone home screen.

5.3 Wi-Fi Provisioning

The project uses the rmaker_app_network component. Upon first boot (or after a long press of the Boot button), the device enters Provisioning Mode, generating a QR code on the serial terminal for secure Wi-Fi credential transfer via Bluetooth Low Energy (BLE).
